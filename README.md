# HiVibe

**File extension:** `.hvibe`

A high-level, structured and maintainable Domain Specific Language for vibe coding, built on top of JSON.

## Motivation

We have entered the era of AI, with powerful tools that are profoundly transforming how we build software. Yet without clear directives, AI can still struggle to fully grasp intent, often producing generic, naive, or suboptimal code.

On large projects, it tends to drift away from the original intent, losing the conceptual thread across iterations.

Vibe-coded projects are especially hard to maintain: often generated in a single session, they become opaque and difficult to extend or revisit.

In AI-driven and vibe coding workflows, the core challenge is not only creating, but preserving intent and coherence over time.

HiVibe addresses this by providing a structured and modular format that keeps projects readable, maintainable, and easily extensible, whether for a developer or an AI model. Each feature is isolated and described, dependencies are explicit, and the full project can be understood, extended, or reorganized without losing meaning.
